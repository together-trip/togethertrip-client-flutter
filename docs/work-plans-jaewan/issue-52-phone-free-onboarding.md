# Issue 52 전화번호 없는 가입 온보딩 구현 계획

## 목표

- 서버의 전화번호 없는 OAuth 계약에 맞춰 앱 가입 흐름을 단순화한다.
- 가입 필수 프로필을 닉네임으로 최소화하고 프로필 이미지는 선택값으로 유지한다.
- 약관 저장, 프로필 저장, 실패 후 재시도 흐름을 보존한다.

## 구현 범위

1. 인증 계약
   - `AuthLoginResult`에서 `temporaryToken`과 `PHONE_VERIFICATION_REQUIRED` 처리를 제거한다.
   - `PROFILE_REQUIRED`와 `AUTHENTICATED` 응답에 포함된 access/refresh token만 저장한다.
2. 가입 UI
   - 전화번호·인증번호 입력, 타이머와 검증 로직을 제거한다.
   - 성별·생년월일 입력과 프로필 완료 조건을 제거한다.
   - 닉네임, 닉네임 중복 확인, 선택 프로필 이미지만 유지한다.
3. 사용자 모델과 API
   - 전화번호 마스킹·인증 상태 모델을 제거한다.
   - 프로필 수정 요청에서 닉네임과 선택 프로필 이미지만 전송한다.
4. 개인정보 및 테스트
   - 앱 내 개인정보 처리방침에서 전화번호·성별·생년월일 수집 문구를 제거한다.
   - 신규 가입, 필수 약관, 중복 확인, 저장 중복 방지, 실패 복구, 기존 사용자 복구를 위젯 테스트한다.

## 검증 명령

```bash
flutter analyze
flutter test
```

## 보안 검토

- 앱에서 전화번호를 입력·표시·전송·로컬 저장하는 코드가 남지 않는지 검색한다.
- `PROFILE_REQUIRED`도 인증 토큰이 있는 상태이므로 이후 API는 기존 토큰 갱신 경로를 사용한다.
- 선택 프로필 이미지는 기존 확장자·MIME·파일 시그니처 검증을 유지한다.
