# Issue #42 기록과 소비 장소 선택 UI

## 배경

기록과 소비 작성 화면은 장소명을 텍스트로만 받고 `latitude`, `longitude`를 항상 `null`로 전송한다. 사용자가 글로벌 장소를 검색하거나 지도에서 직접 위치를 지정할 수 있어야 한다.

## 범위

- 공통 `PlaceSelection` 모델과 장소 검색 service
- 장소 자동완성 결과 선택, 지도 marker 이동, 현재 위치, 직접 입력 fallback을 제공하는 공통 picker
- 기록/소비 생성·수정 폼에 선택된 장소명과 좌표 연결
- 기존 텍스트-only 장소와 좌표가 포함된 기존 데이터 편집 호환
- Android/iOS 지도 키 및 foreground 위치 권한 설정

## 제외 범위

- 백그라운드 위치 권한/추적
- 여행 동선 및 지도 목록 화면
- 검색 결과 로컬 영구 저장

## 아키텍처 판단

- feature-first 구조를 유지해 `features/place`에 model/service/screen을 둔다.
- 외부 Places 키는 앱이 아닌 main 서버가 보유한다.
- 지도 SDK 키는 Android package/iOS bundle 제한을 전제로 플랫폼별로 분리한다.
- API 키가 없거나 위치 권한이 거부되어도 직접 입력은 사용할 수 있다.

## TDD 시나리오

1. 장소 service가 자동완성/상세/역지오코딩 응답을 내부 모델로 파싱한다.
2. 검색 결과 선택 시 장소명과 좌표가 폼 입력에 반영된다.
3. 핀 이동 후 역지오코딩 결과가 선택 상태에 반영된다.
4. 현재 위치 권한 거부/서비스 비활성화 시 오류를 안내하고 직접 입력을 유지한다.
5. 기존 좌표가 있는 게시글 편집 시 marker가 복원된다.
6. 직접 입력으로 전환하거나 장소를 지우면 오래된 좌표가 전송되지 않는다.
7. 기록과 소비 multipart 요청에 선택 좌표가 포함된다.

## 보안·개인정보

- foreground 위치 권한은 사용자가 현재 위치 버튼을 누를 때만 요청한다.
- 현재 위치는 picker 초기화/선택에만 사용하며 별도로 저장하지 않는다.
- 선택해 제출한 장소 snapshot은 여행 참여자에게 보일 수 있음을 기존 위치 약관 정책과 일치시킨다.
