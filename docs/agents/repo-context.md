# Repo Context

## 리포 역할

`app`은 TogetherTrip의 iOS/Android Flutter 모바일 앱이다.

## 책임 범위

- 온보딩, 카카오 로그인, 회원가입, 자동 로그인 흐름
- 여행 목록, 여행 생성, 여행 상세, 기록/소비 등록 UI
- 정산 조회, 송금/수금 확인 UI, 마이페이지, 알림함
- API 클라이언트, 상태 관리, 입력 검증, 화면 전환
- 사진 선택/미리보기, 딥링크, 푸시 알림 권한 처리

## 아키텍처 원칙

- feature-first Flutter 구조를 기본으로 한다.
- presentation, application, domain, data 책임을 분리한다.
- UI는 기획의 1차 MVP 범위를 우선한다.
- 앱 내 검색, 신고/차단, 사진 확대/저장, 외부 공유, 다크 모드는 1차 범위가 아니다.
- API 호출은 gateway를 외부 진입점으로 본다.

## 통신 규칙

- `app -> gateway -> main/chat/notification` 흐름을 기본으로 한다.
- 서비스별 DB나 내부 구현에 직접 의존하지 않는다.
- 딥링크는 접근 권한이 확인된 화면으로만 이동해야 한다.

## 핵심 도메인 주의점

- 기록은 언제든 등록 가능하지만 소비는 여행 기간과 정산 상태 제약을 따른다.
- 정산 시작 이후 소비 금액 관련 수정/삭제는 불가하다.
- 자기 자신이 발생시킨 행동에는 알림을 보내지 않는다.
- 동행자, 방장, 탈퇴/퇴장 사용자 표시 정책을 UI에서 혼동하지 않는다.
